Grip Protocol API v0.4 — Stable

Gaming Incident Intelligence platform. Real-time friction detection, player stress indexing, and verified solution matching across PC, console, and handheld platforms.

https://api.gamegrip.cloud
🟠 Postman Collection 🔔 Webhooks ⚠️ Error Codes

Authentication

All endpoints require an API key passed via the X-API-Key header. Get your key from the Developer Portal. Rate limiting applies: 100 requests/minute per key.

⚠️ Error Reference

All errors return a consistent JSON envelope. Use the code field for programmatic handling and message for human-readable context.

Error envelope
{ "error": true, "code": "RATE_LIMIT_EXCEEDED", "message": "Rate limit of 100 requests/minute exceeded. Retry after 12 seconds.", "retry_after": 12 }
HTTP StatusCodeDescription
400 BAD_REQUEST Missing required fields or invalid JSON body
401 UNAUTHORIZED Missing or invalid API key in X-API-Key header
403 FORBIDDEN API key doesn't have permission for this endpoint (plan restriction)
404 NOT_FOUND Game ID or resource not found in the registry
429 RATE_LIMIT_EXCEEDED Too many requests. Check retry_after field (seconds). Limit: 100 req/min
500 INTERNAL_ERROR Server error. Retry with exponential backoff. If persistent, contact support
503 SERVICE_UNAVAILABLE Pipeline is restarting or under maintenance. Typically resolves within 60 seconds
Example — 400 Bad Request
{ "error": true, "code": "BAD_REQUEST", "message": "Missing required field: 'game'. Expected a string with the game name.", "field": "game" }
Example — 429 Rate Limited
{ "error": true, "code": "RATE_LIMIT_EXCEEDED", "message": "Rate limit of 100 requests/minute exceeded. Retry after 12 seconds.", "retry_after": 12 }
WEBHOOK

Grip Bridge — Webhooks

Receive real-time push notifications when incidents are created, updated, or resolved. Configure your webhook URL in the Developer Portal. All payloads are signed with HMAC-SHA256.

Event Types
EventTriggered when
incident.createdA new friction hotspot is detected and scored
incident.updatedPSI changes significantly or severity is re-graded
incident.resolvedCommunity fix verified and incident closed
fix.foundNew fix matched to an existing incident (before full verification)
Webhook payload — incident.created
{ "event": "incident.created", "timestamp": "2026-05-24T06:15:00Z", "data": { "id": 226, "game_name": "Diablo II Resurrected", "category": "gameplay", "severity": "medium", "psi": 80, "summary": "Party invite bug — can't accept invites from 2+ players", "pipeline_status": "enriched" } }
HMAC Signature Verification

Every webhook includes a X-Grip-Signature header. Verify it using your webhook secret (found in the Developer Portal).

Node.js
Python
// Verify Grip Bridge webhook signature (Node.js) const crypto = require('crypto'); function verifyWebhook(body, signature, secret) { const expected = crypto .createHmac('sha256', secret) .update(body) .digest('hex'); return crypto.timingSafeEqual( Buffer.from(signature), Buffer.from('sha256=' + expected) ); } // Express middleware app.post('/webhook/grip', (req, res) => { const sig = req.headers['x-grip-signature']; if (!verifyWebhook(JSON.stringify(req.body), sig, WEBHOOK_SECRET)) { return res.status(401).json({ error: 'Invalid signature' }); } console.log('Event:', req.body.event, req.body.data); res.status(200).json({ received: true }); });
# Verify Grip Bridge webhook signature (Python) import hmac, hashlib def verify_webhook(body: bytes, signature: str, secret: str) -> bool: expected = 'sha256=' + hmac.new( secret.encode(), body, hashlib.sha256 ).hexdigest() return hmac.compare_digest(signature, expected) # Flask example @app.route('/webhook/grip', methods=['POST']) def grip_webhook(): sig = request.headers.get('X-Grip-Signature', '') if not verify_webhook(request.data, sig, WEBHOOK_SECRET): return jsonify(error='Invalid signature'), 401 event = request.json print(f"Event: {event['event']} — {event['data']}") return jsonify(received=True), 200
